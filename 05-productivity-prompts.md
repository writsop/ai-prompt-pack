# ⚡ Productivity & Workflow Prompts (25 Prompts)

---

## Planning & Strategy

### 1. Weekly Planning System
```
Help me plan my week. Here's my context:

Role: [JOB TITLE]
Top 3 priorities this quarter: [LIST]
Carry-over tasks from last week: [LIST]
Meetings already scheduled: [LIST]
Deadlines this week: [LIST]
Energy pattern: [MORNING PERSON/NIGHT OWL]

Create:
1. Monday-Friday daily schedule with time blocks
2. Top 3 tasks per day (most important first)
3. Deep work blocks (2+ hour uninterrupted)
4. Buffer time for unexpected tasks (20%)
5. Weekly review checklist for Friday
6. One thing to say NO to this week
```

### 2. Project Kickoff Planner
```
Create a project plan for [PROJECT NAME].

Objective: [WHAT SUCCESS LOOKS LIKE]
Deadline: [DATE]
Team: [WHO'S INVOLVED]
Budget: [IF APPLICABLE]
Dependencies: [WHAT NEEDS TO HAPPEN FIRST]

Provide:
1. Work breakdown structure (WBS)
2. Milestone timeline
3. RACI matrix (Responsible, Accountable, Consulted, Informed)
4. Risk register (top 5 risks with mitigation)
5. Communication plan
6. Resource allocation
7. Definition of done for each phase
8. Kickoff meeting agenda
```

### 3. OKR Writer
```
Write OKRs for [TEAM/INDIVIDUAL] for [TIME PERIOD].

Role/Team: [WHO]
Company objective: [HIGHER-LEVEL GOAL]
Current challenges: [WHAT'S NOT WORKING]
Resources available: [TEAM SIZE, BUDGET, TOOLS]

Create 3 Objectives, each with 3-4 Key Results:
- Objectives should be qualitative and inspiring
- Key Results should be quantitative and measurable
- Include stretch targets (70% achievement = success)
- Add initiatives (actions) under each KR
- Include leading and lagging indicators
```

### 4. Decision Framework
```
Help me make this decision: [DESCRIBE DECISION]

Options:
1. [OPTION A]
2. [OPTION B]
3. [OPTION C - if applicable]

Context:
- Timeline: [URGENCY]
- Reversibility: [CAN I UNDO THIS]
- Stakes: [WHAT'S AT RISK]
- Who's affected: [STAKEHOLDERS]

Analyze using:
1. Pros/cons matrix (weighted)
2. Second-order consequences
3. Regret minimization (which would I regret NOT doing)
4. 10/10/10 framework (how will I feel in 10 min/10 months/10 years)
5. Pre-mortem (imagine each option failed - why?)
6. Clear recommendation with reasoning
```

### 5. SWOT Analysis
```
Conduct a SWOT analysis for [BUSINESS/PROJECT/DECISION].

Context: [DESCRIBE SITUATION]
Industry: [SECTOR]
Timeframe: [PERIOD]

For each quadrant provide 5-7 items:

Strengths (internal, positive):
- What do we do better than competitors?

Weaknesses (internal, negative):
- Where do we fall short?

Opportunities (external, positive):
- What trends can we capitalize on?

Threats (external, negative):
- What could hurt us?

Then create a strategy matrix:
- SO strategies (use strengths to capture opportunities)
- WO strategies (overcome weaknesses using opportunities)
- ST strategies (use strengths to counter threats)
- WT strategies (minimize weaknesses and avoid threats)
```

## Communication

### 6. Meeting Agenda Generator

```
Create a meeting agenda for [MEETING TYPE].

Meeting purpose: [WHY]
Attendees: [WHO AND ROLES]
Duration: [MINUTES]
Format: [IN-PERSON/VIRTUAL/HYBRID]

Agenda:
1. Opening (state purpose, expected outcomes)
2. [TOPIC 1] - [TIME] - [PRESENTER] - [GOAL: inform/discuss/decide]
3. [TOPIC 2] - [TIME] - [PRESENTER] - [GOAL]
4. Action items and owners

Also provide:

- Parking lot for off-topic items
- Post-meeting email template
```

### 7. Difficult Conversation Script
```
Help me prepare for a difficult conversation about [TOPIC].

Situation: [WHAT HAPPENED]
Relationship: [BOSS/DIRECT REPORT/PEER/CLIENT]
Goal: [WHAT OUTCOME DO I WANT]
Their likely perspective: [HOW THEY SEE IT]

Provide:
1. Opening statement (non-confrontational)
2. Key points to make (factual, not emotional)
3. Active listening prompts
4. Responses to likely pushback
5. Bridge phrases (to keep conversation productive)
6. Clear ask/proposal
7. Closing and follow-up plan

Tone: Direct but empathetic. Focus on behavior and impact, not character.
```

### 8. Status Update Writer
```
Write a status update for [PROJECT/WORK].

Audience: [MANAGER/STAKEHOLDERS/TEAM]
Period: [WEEK/MONTH/QUARTER]

Raw information:
- Completed: [LIST]
- In progress: [LIST]
- Blocked: [LIST]
- Metrics: [ANY NUMBERS]
- Risks: [CONCERNS]

Format as:
1. TL;DR (3 bullet executive summary)
2. Highlights (wins and progress)
3. Key metrics with trend arrows
4. Blockers and asks (what you need from them)
5. Next period priorities
6. Risks and mitigation

Keep it under 300 words. Lead with impact, not activity.
```

### 9. Feedback Generator
```
Help me write feedback for [PERSON'S ROLE].

Context: [PERFORMANCE REVIEW/360/INFORMAL]
Relationship: [YOUR ROLE TO THEM]
Strengths observed: [LIST]
Areas for growth: [LIST]
Specific examples: [SITUATIONS]

Write feedback using SBI framework (Situation-Behavior-Impact):
1. 3 strengths with specific examples
2. 2 development areas with actionable suggestions
3. Overall summary
4. Forward-looking goals

Tone: Constructive, specific, balanced. Avoid vague praise or criticism.
```

## Personal Productivity

### 11. Morning Routine Designer

```
Design a morning routine for [GOAL].

Wake-up time: [TIME]
Leave for work: [TIME] (or start work if remote)
Current routine: [DESCRIBE]
Goals: [ENERGY/FOCUS/FITNESS/LEARNING]
```

### 15. Time Audit Template
```
Help me audit and optimize how I spend my time.

Role: [JOB]
Hours per week: [NUMBER]
Main frustration: [WHAT EATS YOUR TIME]

Create:
1. Time tracking template (30-min blocks, 5 days)
2. Category system (deep work, admin, meetings, email, breaks)
3. Analysis framework after tracking:
   - Where does time actually go vs where should it go
   - Energy mapping (high energy tasks vs low)
   - Delegation candidates
   - Elimination candidates
   - Automation opportunities
4. Ideal week template based on analysis
5. Implementation plan to shift toward ideal
```

## Job & Career

### 16. Resume Bullet Point Writer
```
Rewrite my resume experience using the STAR method.

Role: [JOB TITLE]
Company: [NAME]
Duration: [DATES]

Raw responsibilities and achievements:
[PASTE YOUR CURRENT BULLET POINTS]

Rewrite each bullet point to:
- Start with a strong action verb
- Include quantifiable results (numbers, percentages, dollar amounts)
- Show impact, not just duties
- Be concise (one line, under 20 words)

Also suggest:
- 3 missing achievements I should add
- Keywords for ATS optimization for [TARGET ROLE]
- Skills section recommendations
```

### 17. Cover Letter Generator
```
Write a cover letter for [POSITION] at [COMPANY].

Job posting key requirements:
1. [REQUIREMENT]
2. [REQUIREMENT]
3. [REQUIREMENT]

My relevant experience:
1. [EXPERIENCE]
2. [EXPERIENCE]
3. [EXPERIENCE]

Why this company: [GENUINE REASON]

Structure:
- Opening: Hook with a specific connection to the company
- Body 1: Match #1 requirement to my experience (with result)
- Body 2: Match #2 requirement (with result)
- Body 3: Cultural fit / unique value I bring
- Close: Confident CTA, not desperate

Tone: Professional, authentic, not generic. Under 350 words.
```

## Automation & Systems

### 21. Standard Operating Procedure
```
Create an SOP for [PROCESS].

Process: [NAME]
Purpose: [WHY THIS EXISTS]
Frequency: [HOW OFTEN]
Owner: [WHO'S RESPONSIBLE]
Tools used: [LIST]

SOP Structure:
1. Purpose and scope
2. Prerequisites (what's needed before starting)
3. Step-by-step instructions (numbered, with screenshots placeholders)
4. Decision points (if/then branches)
5. Quality checks at each stage
6. Common errors and troubleshooting
7. Escalation procedure
8. Version history

Write for someone doing this for the first time. No assumed knowledge.
```

```
4. Setup time estimate
5. Ongoing maintenance
```

### 24. Delegation Planner
```
Help me decide what to delegate and how.

My role: [TITLE]
My team: [WHO REPORTS TO ME]
My current tasks: [LIST ALL MAJOR TASKS]
Hours per week: [TOTAL]

For each task, assess:
1. Can ONLY I do this? (unique expertise required)
2. Should I do this? (highest and best use of my time)
3. Who could do this? (team member match)
4. What training is needed?
5. What does successful delegation look like?

Create:
- Keep list (tasks only I should do)
- Delegate list (with who and transition plan)
- Eliminate list (tasks nobody should do)
- Automate list (tasks a tool should handle)
```
